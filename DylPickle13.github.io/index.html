EnGem

EnGem is a local assistant and automation framework that connects a streaming LLM to host-side tooling and exposes a Discord bot interface. It provides:

  • LLM orchestration and sub-agent execution plans
  • A Discord bot for text chat, scheduling, and execution-plan progress tracking
  • Persistent semantic memory backed by ChromaDB
  • Modular skills for browser automation, running code, notebooks, and media generation
  • Output capture and artifact relocation into generated_files/

Quick start

1. Create and activate a virtual environment, then install dependencies:

python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt

2. Install Playwright browser binaries (required for browser automation):

python -m playwright install

3. Set required environment variables (examples):

  • DISCORD_BOT_TOKEN — Discord bot token.
  • PAID_GEMINI_API_KEY — Gemini API key used by the bot and skills.
  • DISCORD_ALLOWED_CHANNELS — Comma-separated allowed channel names.

4. Start the Discord bot from the repository root:

python discord_bot.py

Entry points & core modules

  • discord_bot.py — Main bot entrypoint. Handles text messages, cron and heartbeat jobs.
  • llm.py — LLM orchestration: intent classification, manager, sub-agent dispatch.
  • memory.py — ChromaDB-backed semantic memory store.
  • computer_use.py & skills/use_browser.py — Browser automation using Playwright.
  • skills/run_python.py — Run Python snippets in a subprocess.
  • skills/run_notebook.py — Execute notebooks via Papermill.
  • skills/generate_image.py / skills/generate_video.py — Media generation.

Data & storage

  • Conversation histories: memory/channel_history/*.md.
  • Vector DB: memory/vector_db/ (ChromaDB persistent client).
  • Generated outputs: generated_files/.
  • Agent instruction templates: agent_instructions/.

Notes & troubleshooting

  • Playwright requires browser binaries; run python -m playwright install.
  • LLM API calls are wrapped with exponential backoff to handle transient errors.
  • ChromaDB data lives under memory/vector_db/. Back up to preserve memories.

Developer notes

  • Skills are lightweight Python modules under skills/ and are automatically discovered.
  • Generated artifacts from skill runs are collected in generated_files/.
  • For development, run the bot locally after setting the required env vars and installing dependencies.
